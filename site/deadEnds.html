Dead Ends

8-player Unity horror game • P2P networking • Proximity voice chat

A multiplayer horror game built in Unity. I designed and implemented core gameplay systems, peer-to-peer networking, and proximity-based voice chat with a focus on responsiveness, stability, and clear player feedback.

Play on Itch.io

  • Focus: Multiplayer systems, real-time gameplay, UX clarity
  • Engine: Unity (C#)

  • Git collaboration

How I Built This

Networking Architecture

  • Designed a P2P session flow: lobby → host selection → join/leave handling
  • Implemented state replication for player movement and key interactions
  • Added interpolation/smoothing so remote players feel stable under latency
  • Built resilient message handling (validation, sequencing, and edge cases)

Proximity Voice Chat

  • Integrated proximity-based attenuation (distance affects volume)
  • Ensured voice state stays consistent across joins/leaves
  • Focused on UX clarity: who you can hear, when, and why
  • Tested under real play sessions to tune ranges and mixing

Gameplay Systems

  • Core interaction loop: hiding, seeking, traversal, player feedback
  • Systemized UI + prompts so interactions are readable during panic moments
  • Iterated quickly using debug toggles and in-editor tools

Stability & Debugging

  • Added logging hooks for networking events to reproduce desync issues
  • Created stress tests for join/leave and rapid state changes
  • Tracked down race conditions and “late join” inconsistencies
